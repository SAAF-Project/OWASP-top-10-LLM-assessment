# LLM05 — Improper Output Handling

## Description
Improper Output Handling occurs when LLM-generated content is passed to downstream systems or rendered in interfaces without sufficient validation or sanitisation. This can lead to XSS, SSRF, code injection, or other downstream attacks.

## Risk
- XSS if model output is rendered as HTML without escaping
- SQL injection if model output is embedded in database queries
- Code injection if model output is executed via eval() or subprocess
- SSRF if model-generated URLs are fetched without validation

## Key Assessment Questions
1. Is model output sanitised before being rendered in a web interface?
2. Is model output validated before being passed to system calls, database queries, or code execution?
3. Are there output schemas that constrain model responses to safe formats?
4. Does the agent execute code or system commands generated by the model?
5. Is there a whitelist of allowed output formats/actions?

## Mitigations
- Always escape model output before HTML rendering
- Never pass model output directly to SQL queries, shell commands, or eval()
- Use structured output (JSON schemas) to constrain model responses
- Validate URLs and paths in model output before fetching/accessing
- Apply content security policies on any web-facing interface
